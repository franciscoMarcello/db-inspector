description 2. Template de saída (opcional)

Guia para criar Template PDF

Use template PDF quando o relatório precisa sair em formato fixo, com cabeçalho, totais, agrupamentos ou layout de impressão.

Como o template recebe os dados
  1. A SQL do relatório define as colunas disponíveis para o template.
  2. Cada alias da SQL vira um campo no template, por exemplo nome_produtor.
  3. No Jasper, use os campos como $F{nome_produtor}.
  4. Parâmetros detectados na SQL podem ser usados como filtros do relatório.
  5. O template deve ser compatível com as colunas retornadas pela SQL principal.
Boas práticas
  1. Use aliases simples, sem espaço e sem acento: data_producao, quantidade.
  2. Evite mudar nomes de aliases depois que o template estiver pronto.
  3. Formate valores monetários, datas e números no template quando possível.
  4. Para totais, prefira trazer o valor agregado pela SQL ou criar variável no Jasper.
  5. Teste a SQL antes de vincular o template para garantir que os campos existem.
Exemplo de SQL para template
SELECT
  pd.dt AS data_producao,
  p.nome AS produtor,
  d.nome AS dieta,
  SUM(pd.qt_ordem_realizada) AS quantidade_realizada
FROM confinamento.producao_dieta pd
JOIN confinamento.produtor p ON p.id = pd.produtor_id
JOIN confinamento.dieta d ON d.id = pd.dieta_id
WHERE pd.dt >= :data_inicial
  AND pd.dt <= :data_final
GROUP BY pd.dt, p.nome, d.nome
ORDER BY pd.dt, p.nome
No template Jasper, os campos desse exemplo seriam usados como $F{data_producao}, $F{produtor}, $F{dieta} e $F{quantidade_realizada}.
Se o PDF sair em branco ou com erro de campo, verifique se o nome do campo no template é exatamente igual ao alias retornado pela SQL.

compare_arrows 4. Comparação SAP HANA (opcional)

Preencha apenas se este relatório precisa ser comparado com dados do SAP HANA.

Guia para criar SQL de comparação

A comparação cruza o resultado do Agromobi com o SAP HANA usando uma coluna de chave.

Regras obrigatórias
  1. A SQL principal e a SQL SAP devem retornar uma coluna com o mesmo alias da chave.
  2. O campo comparisonKey deve ser exatamente esse alias, por exemplo id_agromobi.
  3. Campos que serão comparados precisam ter aliases iguais nas duas consultas.
  4. Use os mesmos placeholders de filtro nas duas consultas quando o filtro afetar os dois lados.
  5. Evite SELECT *; retorne apenas chave e colunas necessárias para comparar.
Boas práticas
  1. Para números, retorne número puro, não texto formatado com TO_CHAR.
  2. Para datas, use o mesmo formato ou o mesmo tipo dos dois lados.
  3. Use COALESCE quando nulos puderem atrapalhar a comparação.
  4. No SAP HANA, use aspas duplas em tabelas/colunas com nomes sensíveis a maiúsculas.
  5. No SAP HANA, também coloque os aliases entre aspas duplas para preservar o nome retornado.
  6. Ordenação é opcional; a comparação usa a chave, não a ordem das linhas.
Exemplo Agromobi
SELECT
  pd.id_agromobi AS id_agromobi,
  pd.dt AS data_producao,
  SUM(pd.qt_ordem_realizada) AS quantidade
FROM confinamento.producao_dieta pd
WHERE pd.dt >= :data_inicial
  AND pd.dt <= :data_final
GROUP BY pd.id_agromobi, pd.dt
Exemplo SAP HANA
SELECT
  T0."U_IdAgromobi" AS "id_agromobi",
  T0."DocDate" AS "data_producao",
  SUM(T1."Quantity") AS "quantidade"
FROM "OINV" T0
JOIN "INV1" T1 ON T1."DocEntry" = T0."DocEntry"
WHERE T0."DocDate" >= :data_inicial
  AND T0."DocDate" <= :data_final
GROUP BY T0."U_IdAgromobi", T0."DocDate"
Neste exemplo, informe id_agromobi no campo comparisonKey. A tela vai comparar data_producao e quantidade para cada chave.
Se aparecer erro de campo de comparação ausente, significa que uma das duas consultas não retornou a coluna com o alias informado em comparisonKey.
